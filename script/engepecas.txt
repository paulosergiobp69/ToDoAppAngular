		--> $fillable: No eloquent ORM, o atributo $ fillable é um array contendo todos aqueles campos da tabela que podem ser preenchidos usando atribuição em massa. 
		[Atribuição em massa refere-se ao envio de uma matriz para o modelo para criar diretamente um novo registro no Banco de Dados
		--> O preenchimento é obrigatório no Laravel?
		A instância de modelo inserida será retornada a você pelo método: ... 
		No entanto, antes de usar o método de criação, você precisará especificar uma propriedade preenchível 
		ou protegida em sua classe de modelo. 
		Essas propriedades são necessárias porque todos os modelos do Eloquent são protegidos 
		contra vulnerabilidades de atribuição em massa por padrão.

		-->  o atributo $ hidden é puramente para quando o modelo é convertido em uma matriz ou convertido em JSON, 
		portanto, não deve afetar as inserções (ou qualquer outra coisa) 
		
		c) $cast --> deixa
		--> A propriedade $ casts em seu modelo fornece um método conveniente de conversão de atributos em tipos de dados comuns. 
		A propriedade $ casts deve ser uma matriz onde a chave é o nome do atributo sendo convertido, 
		enquanto o valor é o tipo para o qual você deseja converter a coluna.
